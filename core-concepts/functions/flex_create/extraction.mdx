title: "FLEX_CREATE - Extraction"
---

For conceptual information about Extraction, see [Extraction](https://flexor.mintlify.app/core-concepts/extraction).

## Overview

`FLEX_CREATE` creates a persistent column in your table and activates a recurring pipeline that automatically processes new rows. Unlike `FLEX`, which runs ad-hoc queries, `FLEX_CREATE` stores results and incrementally updates them as new data arrives.

## Syntax

```sql
FLEX_CREATE('<classification_input> -> <extraction_input>', '<column_name>', '<table_name>');
```

## Parameters

| Parameter          | Description                                                          |
| ------------------ | -------------------------------------------------------------------- |
| `extraction_input` | A classification input followed by an Extraction input               |
| `column_name`      | The name of the column to create in the output table.                |
| `table_name`       | The table within the Flexor schema to execute the transformation on. |

## Output columns

When you run `FLEX_CREATE` with an extraction input, the following columns are created:

| Column                   | Type  | Description                                                                                                              |
| ------------------------ | ----- | ------------------------------------------------------------------------------------------------------------------------ |
| `<column_name>`          | Array | Extracted values. Contains `["NA"]` if the boolean is False or no entity is found. Contains `[""]` if not yet processed. |
| `<column_name>_evidence` | Array | Raw text snippets corresponding to each extraction.                                                                      |

## Example

The following example creates a persistent column to extract products from return requests:

```sql
SELECT
    FLEX_CREATE(
        'Did the customer ask to return the product? -> product',
        'returned_product',
        '<table_name>'
    );
```

This creates two columns in your output table:

- `returned_product` — Array of extracted product names.
- `returned_product_evidence` — Array of supporting evidence text.
